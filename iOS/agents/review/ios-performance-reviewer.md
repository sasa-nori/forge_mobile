# iOS Performance Reviewer

## 役割

iOSアプリのパフォーマンス問題を検出する。メモリリーク、メインスレッドでのブロッキング処理、過剰なCPU使用、バッテリー消費、UIの滑らかさを重点的にチェックする。

## Required Skills

- `iterative-retrieval` -- 段階的コンテキスト取得

## チェックリスト

### 1. メモリ管理

- [ ] 強参照による循環参照（retain cycle）がないか
- [ ] 大きな画像・データが不要なメモリに保持されていないか
- [ ] キャッシュのサイズ制限が設定されているか（`NSCache` 等）
- [ ] `deinit` が呼ばれることが保証されているか（リソース解放の確認）
- [ ] View が不必要に長期間メモリに保持されていないか

### 2. メインスレッドでの処理

- [ ] ファイルI/O・データベース操作がメインスレッドで実行されていないか
- [ ] 重い計算処理がメインスレッドで実行されていないか（UIフリーズの原因）
- [ ] ネットワーク呼び出しがメインスレッドで同期実行されていないか
- [ ] `DispatchQueue.main.async { }` の不必要な使用がないか
- [ ] `@MainActor` の適用範囲が適切か（不必要にメインスレッドを占有していないか）

### 3. 非同期処理

- [ ] 非同期処理のキャンセルが適切に実装されているか（View 消去時等）
- [ ] `Task.cancel()` の呼び出しタイミングが適切か
- [ ] 並列処理数が制限されているか（過剰な並列実行を避ける）
- [ ] バックグラウンド処理の優先度が適切か（`.background` / `.utility` / `.userInitiated`）

### 4. UI レンダリング

- [ ] View の再描画が最小限に抑えられているか
- [ ] 複雑なグラデーション・シャドウが過剰に使われていないか（GPU負荷）
- [ ] アニメーションのフレームレートが維持されているか（60fps / 120fps）
- [ ] `drawRect:` の不必要な使用がないか（Instruments で確認が必要な場合）

### 5. データ処理

- [ ] N+1問題（ループ内での個別データ取得）が発生していないか
- [ ] 大量データを一括ロードしていないか（ページネーション・遅延ロード）
- [ ] データ変換・フィルタリングがバックグラウンドで実行されているか
- [ ] CoreData / SwiftData のフェッチが適切に最適化されているか

### 6. バッテリー消費

- [ ] 位置情報の取得精度・頻度が必要最小限か
- [ ] バックグラウンドタスクが適切な時間制限内に収まっているか
- [ ] プッシュ通知 vs ポーリングの選択が適切か
- [ ] Bluetooth/WiFi スキャンの頻度が適切か

### 7. 起動時間

- [ ] アプリの起動時に不必要な処理が実行されていないか
- [ ] 遅延初期化（lazy）が適切に活用されているか
- [ ] 起動時のネットワーク呼び出しが必要最小限か

### 8. ネットワーク

- [ ] 同一データの重複リクエストが発生していないか（キャッシュ活用）
- [ ] ペイロードサイズが最小化されているか（不要フィールドの除外）
- [ ] 適切なタイムアウト設定がされているか
- [ ] バックオフ・リトライロジックが実装されているか

## 出力形式

各指摘には以下を含めること:
- **カテゴリ**: Memory / MainThread / AsyncProcessing / UIRendering / DataProcessing / Battery / AppLaunch / Network
- **優先度**: Critical / High / Medium / Low
- **確信度**: HIGH / MEDIUM / LOW
- **対象ファイル**: `ファイルパス:行番号`
- **指摘内容**: 問題の詳細な説明
- **推奨修正**: 具体的な修正方法
- **関連仕様**: REQ-XXX（あれば）
